Level 10 — Orientation

The rotation trick is readable at first contact: the portrait gap is only 18 pt while Bit is 22 pt wide (lines 39–40), so the corridor visibly rejects the player, and the t=0 discovery text plus de-spoiled panel (lines 1129–1137, 1178) nudge without spoiling the solution. Reach is a non-issue because the route is a flat walk from spawn (x = 170) to the exit (x = 740) with no vertical jumps, and the landscape gap of 100 pt sits well inside the 130 pt horizontal budget. Hint pacing is broken: the crusher kills in roughly 5–7 seconds, yet the explicit hint from `hintText()` is gated behind the base ~18–22 second no-progress timer (line 1550). Struggling players therefore die several times before any earned help arrives, and `notePlayerStruggle()` on death (line 1514) only feeds back the same single fallback string instead of escalating guidance. Code quality is rough: `updateCorridorPhysics()` is dispatched asynchronously after a 0.5 s delay (line 1279) while the wall nodes animate over the same interval (lines 1266–1271), leaving a half-second window where visible walls and collision bodies disagree. The non-animated path still uses the async dispatch, so initial orientation can also start with stale bodies, and `isWideCanvas` relies on brittle hard-coded size thresholds (`min(size.width, size.height) >= 700` at line 89 and `size.height > 1000` at line 142). The fix is to call `updateCorridorPhysics()` synchronously whenever `corridorGap` changes and animate only the decorative visuals, eliminating desynced collision.
