Level 13 — WiFi

The t=0 panel "THE SIGNAL COMES AND GOES..." (line 747) never names WiFi or Control Center, so first-timers can wander without knowing the input. Platforming reach is generally safe: the iPhone chasm is intentionally un-jumpable at 232 logical px and only bridged by WiFi-ON stones (lines 220-222), while the +55 and +45 vertical hops in Segment B (lines 240, 249) stay inside the 85 pt rise budget and the iPad route respects the 130 pt edge-to-edge gap. The hinting, however, is broken: there is no `notePlayerStruggle()` override, and `hintText()` (line 924) always returns the same one-liner, so stuck players get only the base-class 18-22 s fallback with no escalating guidance. Code quality is also weak: the `downloadProgress` bar is purely cosmetic (lines 666-668), contradicting LEVEL-GUIDE's claim that completing the download unlocks the final section and creating a false objective. `isWideCanvas` (line 54) hard-codes `size.height > 1000 && size.width > 430`, which can mis-gate split-view or compact-height iPad layouts. One concrete improvement: implement escalating struggle hints (e.g., "Try toggling WiFi", then "Signal bridges vanish when WiFi is off", then "The wall opens when the signal drops") and either remove the misleading download bar or make it actually gate the exit.
