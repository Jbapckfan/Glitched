Level 3 — Static

The mic/noise-shield mechanic is introduced clearly enough, but the opening panel’s “MAKE NOISE TO BLOCK LASERS” (lines 1014/1021) only teaches half the rule. The inverse behavior is left to the dashed laser’s in-world placard “THIS ONE LISTENS DIFFERENTLY.” (line 773), which is fair as a t=0 non-spoiler yet too subtle to read reliably during motion. Platforming reach is technically within budget—iPhone treads overlap horizontally and rises stay under 25 points, while iPad edge gaps land at 84–100 pt and vertical steps around ±1—but the iPhone side is so safe it plays like a staircase, not a laser-dodge challenge. Hints are underdeveloped: `hintText()` at line 1272 belatedly clarifies the inverse rule, yet there is no `notePlayerStruggle()` override anywhere, so repeated deaths only trigger the base-class fallback instead of escalating, level-specific guidance. The code carries several rough edges: the `isWideCanvas` check `size.height > 1000 && size.width > designSize.width` (line 48) wrongly excludes landscape iPad from the composed tablet layout; `instructionPanel!` is force-unwrapped on `addChild` (line 978); and font names “Menlo-Bold” and “Menlo” are hardcoded (lines 774/1015/1022) rather than using `VisualConstants.Fonts`. One concrete fix is to remove the width condition from `isWideCanvas` so landscape iPads actually use the hand-authored climb, tighten the iPhone platform fractions to eliminate overlapping treads, and add a `notePlayerStruggle()` override that nudges the player toward silence after the first death at the final barrier.
