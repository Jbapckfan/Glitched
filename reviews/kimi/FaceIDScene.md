Level 19 — Face ID

The t=0 instruction panel spoils the vault by saying "WALK TO THE VAULT" and "TAP IT TO SCAN YOUR FACE" before the player even reaches it, and it never hints that the level needs three scans. After the first scan, the vague "TAP THE NEXT GATE" prompt gives no indication the same second door must be tapped twice more, and because `isUnlocked` stays false until step 3 the exit nudge "ONE MORE SCAN" points at an already-open gate. Platforming is safe: iPhone gaps are tiny, and the iPad route keeps edge-to-edge gaps ≤90 and top-to-top rises clamped at ≤85. The hint system is flat: `hintText()` only returns "Authenticate identity" and there is no `notePlayerStruggle()` override, so the base class can only repeat the same phrase. Code rough edges include dead `faceFrameWidth` assignments in both build paths (lines 207, 308) that `createVaultDoor()` ignores by hardcoding width 50 (line 477), and the unlock mismatch where `advanceScanStep()` case 2 removes the second-door blocker (lines 803–824) yet `handleExit()` rejects the player until case 3 sets `isUnlocked` (line 954). The most concrete fix is to align the unlock flow with the UI by setting `isUnlocked = true` in case 2 and adding a struggle-counted hint ladder from "Tap the glowing vault" to "Two scans are required" to the on-screen ID fallback.
