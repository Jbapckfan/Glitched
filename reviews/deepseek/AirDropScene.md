Level 28 — Share to Decode
The share-to-decode mechanic is clever, but first-encounter fairness is weak because the instruction panel's "THE CODE IS SCRAMBLED" line doesn't tell players that sharing is the decode action—they may stare at the terminal, not realizing the SHARE button is the key. The completability math is sound: the locked door body is 120pt tall, well above Bit's ~91pt jump apex, so the door cannot be cleared without unlocking. The hint quality is excellent: hintText() explicitly names the SHARE button and the keypad, and the fallback path (AccessibilityManager.needsFallbackUI) reveals the decoded code without bypassing the puzzle entirely. A specific code bug: keySet.formUnion(Set(cipherText)) can produce duplicate symbols if the ciphertext contains the same characters as the answer, which is fine, but the keypad shuffles each time—if a player gets a 12-key grid with both the answer and ciphertext, they might still brute-force it. One concrete improvement: add a small "DECODED: ????" label that fills in one symbol at a time when the player holds the SHARE button, showing that sharing reveals the code rather than unlocking the door directly.
