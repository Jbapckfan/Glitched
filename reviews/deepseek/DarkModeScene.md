Level 8 — Dark Mode
The dual-platform mechanic (moon=solid in dark, sun=solid in light) is elegantly designed, but first-encounter fairness is compromised by the instruction panel's cryptic "NIGHT AND DAY DON'T AGREE ON WHAT'S REAL" line—players may not realize that system dark mode is the toggle. The completability math is sound: the door platform is one tier above the rest-ledge (107.5pt rise vs ~91pt apex), so the jump can't be skipped, and the dark-only platforms require toggling dark mode to proceed. The hint quality is excellent: the discoverability nudge ("THAT MOON ISN'T DECOR") appears after 7 seconds, and hintText() explicitly names the dark mode toggle and the dark-locked door. A specific code bug: hasReceivedAppearanceInput is set to true on any appearance change, but if the player uses the in-scope fallback toggle, the real system dark mode state diverges from the game's internal state—the game should re-sync with the system whenever the fallback is used. One concrete improvement: add a small "DARK" / "LIGHT" label above the moon sensor on the door platform, showing the current mode and subtly hinting that this state matters for the door.
