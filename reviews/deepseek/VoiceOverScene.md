Level 27 — VoiceOver / Accessibility
The "phase the path in" mechanic is a masterful accessibility-first design, but first-encounter fairness is compromised because the instruction panel's "THE GAP DOESN'T CARE THAT YOU SEE IT" line doesn't tell players how to phase the path—they may tap the screen repeatedly, expecting a button. The completability math is sound: real stones are 40pt wide, decoys are visually distinct (barred glyphs), and real stones gain physics bodies only when pathPhasedIn is true, so the gap is uncrossable until the player engages the mechanic. The hint quality is excellent: hintText() explicitly names VoiceOver and the bottom-left accessibility button, and the death hint ("BARRED TILES AREN'T REAL") escalates after repeated failures. A specific code bug: systemReduceMotion is read but never used to suppress the decoy pulse animation—showRevealedState() should check it and set decoys to a static low alpha instead of pulsing. One concrete improvement: add a small, persistent "PHASE PATH" button in the bottom-left corner (next to the existing accessibility circle) that appears after 3 deaths without phasing, giving players who can't enable VoiceOver a clear, direct way to proceed.
