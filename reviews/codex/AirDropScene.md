Level 28 — System share sheet

The share-code puzzle is clear and fair: the terminal shows only ciphertext plus a rotation rule, and the actual plaintext arrives through the system share sheet or fallback. There is no meaningful jump pressure beyond the locked door, which is tall enough to prevent a bypass, and the iPad staircase stays within the 130-gap and 85-rise limits. Deaths and wrong keypad submissions call `notePlayerStruggle()`, while revealing the code records progress, so hint escalation covers both movement and puzzle errors. The t=0 "SHARE TO DECODE" button is explicit, but that is probably necessary because the mechanic lives outside the game. A rough edge is cancellation feedback: if the share sheet is dismissed without completion, nothing in the scene appears to explain that no code was decoded. One concrete improvement: set a terminal status like "SHARE CANCELED - TRANSMISSION STILL ENCRYPTED" when the completion handler reports `completed == false`.
