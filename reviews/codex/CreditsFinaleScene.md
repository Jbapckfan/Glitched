Level 30 — None (victory lap)

As a credits climb, the mechanic is intentionally plain, and "CLIMB THE CREDITS / WATCH OUT FOR BUGS" is fair but not subtle. The route is designed as a vertical credits ladder with iPad rises clamped to the safe 85-point budget and no obvious unjumpable gaps. Deaths call `notePlayerStruggle()`, and the hint repeats the climb/bug objective, so stuck players get at least basic direction. The rough pacing issue is that `handleExit()` succeeds the level before a long victory fake-out sequence that eventually loads Level 31; if the app is killed mid-sequence, completion state and actual narrative position can diverge. A code rough edge is the force unwrap `centers.last!` in the composed finale setup, which is probably safe but unnecessary in a generated layout. One concrete improvement: persist an explicit "credits sequence reached Level 31 handoff" flag or shorten the handoff so success and next-level load are closer together.
