Level 23 — Battery level reading

The "running low" clue is fair and the battery readout makes the state visible, but the permanent drain button turns a real battery mechanic into an in-game knob. The platform plan is sound: lower battery removes high stones and activates the hidden exit, and the iPad route describes safe single-tier rises. Deaths call `notePlayerStruggle()`, and the hint tells the player to lower battery percentage, so the escalation is direct. The fake-exit taunt is useful because it prevents players from assuming a collision bug when the visible door fails. The code rough edge is called out in the file itself: `updateBatteryState()` can recreate stone physics bodies at a default 45-point width after some stones were resized, creating a mismatch between visual and collision width. One concrete improvement: store each stepping stone's realized width and rebuild its physics body from that value whenever battery state changes.
