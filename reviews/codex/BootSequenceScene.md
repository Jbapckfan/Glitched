Level 0 — Drag interaction

The drag mechanic is extremely clear on first encounter: the boot log says "DRAG TO 100% TO BOOT" and the post-boot handle says "DRAG TO COMPLETE", so fairness is high but discovery is basically zero. That is acceptable for a boot/tutorial scene, but it means the level teaches compliance more than it teaches curiosity. There are no platform gaps or rises, so Bit's 130-gap and 85-rise budgets are not relevant here. The clue path is weak after the first moment because this scene does not override `hintText()` or feed `notePlayerStruggle()`, so a player who misses the handle only gets generic idle behavior. A rough code edge is the duplicated `backgroundColor = .black` assignment in the reduced-motion completion branch, which is harmless but sloppy in a scene that should be foundational. One concrete improvement: add a boot-specific idle hint and pulse or enlarge the draggable handle after a few seconds without touching it.
