Level 24 — App cache / storage

The storage-space puzzle is fairer than a one-button cache clear because the player must first reach and arm the purge terminal. The jump route is explicitly budgeted, with a 60-point phone rise and an iPad vertical column built from safe one-tier steps. The t=0 storage/junk messaging points at the idea without immediately saying "go clear cache", and `hintText()` escalates from terminal location to the actual storage action. The problem is that deaths do not call `notePlayerStruggle()`, so a player repeatedly missing the terminal route does not get faster help. The code also removes the cache file in `willMove`, which is intentional for teardown but fragile if scene transitions happen during an unfinished Settings detour. One concrete improvement: call `notePlayerStruggle()` on death and show a terminal arrow after two falls before revealing the storage-clear instruction.
