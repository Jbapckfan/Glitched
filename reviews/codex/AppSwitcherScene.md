Level 18 — App Switcher peek (swipe up to multitask view)

The "look away" clue is strong and does not name the gesture until the hint, so first-contact discovery is fair. Platform reach is safe on both paths, with the iPad climb built as one-tier steps and the hazards supplying the real challenge. Deaths call `notePlayerStruggle()`, and entering peek mode calls `notePlayerProgress()`, which is exactly the right hint contract. The randomized spike phases make the peek mechanic meaningful, though they also reduce the value of learning timing by eye. A rough edge is `isWideCanvas` requiring `size.height > 1000 && size.width > 700`, so some iPad landscape or split-view canvases may fall back to a phone-ish layout with very different framing. One concrete improvement: gate the composed layout with a stable minimum-dimension check or add an explicit non-composed landscape audit path.
