Level 8 — System dark/light mode toggle (Settings > Display & Brightness)

The dark-mode mechanic is clear and fair: the "NIGHT AND DAY" language points at the rule without literally saying "open Settings", and the doors reinforce dark versus light behavior. The route is reachable by construction, with the iPad generated gates using single safe tier steps and the phone door height blocking bypass jumps. Deaths call `notePlayerStruggle()`, and unlocking the door records progress, so the stuck-player hint path is solid. The t=0 copy does reveal that the door "trusts the dark", but that is probably acceptable because the system action is already outside the game. The main rough edge is lifecycle state: the scene mutates `UserDefaults.forceDarkMode`, and a crash or unusual exit before cleanup could leave app-wide mode forcing in the wrong state. One concrete improvement: wrap the forced dark-mode override in a scoped lifecycle helper that restores the previous value from `deinit` and `willMove`.
